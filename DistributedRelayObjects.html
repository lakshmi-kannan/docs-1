DistributedRelay Object¶

config/DistributedRelay¶

  • Multiple objects of this type can exist in a system.
PARAMETER NAME DATA TYPE DESCRIPTION DEFAULT VALID VALUES
DrniName [KEY] string The unique identifier allocated to this Distributed Relay by the local System. This attribute identifies a Distributed Relay instance among the subordinate managed objects of the containing object. N/A N/A
EncapMethod string This managed object is applicable only when Network / IPL sharing by time (9.3.2.1) or Network / IPL sharing by tag (9.3.2.2) or Network / IPL sharing by encapsulation (9.3.2.3) is supported. The object identifies the value representing the encapsulation method that is used to transport IPL frames to the Neighbor Portal System when the IPL and network link are sharing the same physical link. It consists of the 3-octet OUI or CID identifying the organization that is responsible for this encapsulation and one following octet used to identify the encapsulation method defined by that organization. Table 9-11 provides the IEEE 802.1 OUI (00-80-C2) encapsulation method encodings. A Default value of 0x00-80-C2-00 indicates that the IPL is using a separate physical or Aggregation link. A value of 1 indicates that Network / IPL sharing by time (9.3.2.1) is used. A value of 2 indicates that the encapsulation method used is the same as the one used by network frames and that Network / IPL sharing by tag (9.3.2.2) is used 00-80-C2-01 N/A
GatewayAlgorithm string This object identifies the algorithm used by the DR Function to assign frames to a Gateway Conversation ID. Table 9-7 provides the IEEE 802.1 OUI (00 00-80-C2-01 N/A
IntfRef string Read-write Interface Identifier of the Aggregator Port assigned to this Distributed Relay N/A N/A
NeighborGatewayAlgorithm string TThis object identifies the value for the Gateway algorithm of the Neighbor Portal System 00-80-C2-01 N/A
NeighborPortAlgorithm string This object identifies the value for the Port Algorithm of the Neighbor Portal System 00-80-C2-01 N/A
PortalPriority uint16 A 2octet read-write value indicating the priority value associated with the Portals System ID. Also used as the Actors System Priority (6.3.2) for the emulated system. 32768 N/A
IntfReflist string Read-write list of the Interface Identifiers of the Ports to the Intra-Portal Links assigned to this Distributed Relay. Each Interface Identifier N/A N/A
IntraPortalPortProtocolDA string A 6-octet read-write MAC Address value specifying the DA to be used when sending DRCPDUs 01-80-C2-00-00-03 N/A
NeighborAdminDRCPState string A string of 8 bits 00000000 N/A
PortalAddress string A read-write identifier of a particular Portal. Portal-Addr has to be unique among at least all of the potential Portal Systems to which a given Portal System might be attached via an IPL Intra-Portal Link. Also used as the Actors System ID (6.3.2) for the emulated system N/A N/A
PortalSystemNumber uint8 A read-write identifier of this particular Portal System within a Portal. It is the responsibility of the network administrator to ensure that these numbers are unique among the Portal Systems with the same aDrniPortalAddr (7.4.1.1.4) N/A N/A
ThreePortalSystem bool A read-write Boolean value indicating whether this Portal System is part of a Portal consisting of three Portal Systems or not. Value 1 stands for a Portal of three Portal Systems false N/A

FlexSwitch CURL API Supported¶

  • GET By Key

    curl -X GET -H ‘Content-Type: application/json’ –header ‘Accept: application/json’ -d ‘{<Model Object as json-Data>}’ http://device-management-IP:8080/public/v1/config/DistributedRelay

  • GET By ID

    curl -X GET http://device-management-IP:8080/public/v1/config/DistributedRelay/<uuid>

  • GET ALL

    curl -X GET http://device-management-IP:8080/public/v1/config/DistributedRelays?CurrentMarker=<x>\&Count=<y>

  • CREATE(POST)

    curl -X POST -H ‘Content-Type: application/json’ –header ‘Accept: application/json’ -d ‘{<Model Object as json-Data>}’ http://device-management-IP:8080/public/v1/config/DistributedRelay

  • DELETE By Key

    curl -X DELETE -i -H ‘Accept:application/json’ -d ‘{<Model Object as json data>}’ http://device-management-IP:8080/public/v1/config/DistributedRelay

  • DELETE By ID
